src/symbolic/parser.rs — Elastic PEG + Autofix Bridge (v2.6)

//! v2.6: Elastic PEG Parser with Autofix Bridge. //! //! v2.5 introduced a strict formal PEG parser (pest). v2.6 adds a //! "Syntax Physician" fallback: if the PEG grammar rejects an expression //! that is *mathematically* sound but syntactically non-standard //! (e.g. `grad(f)` vs `∇f`), a low-temperature LLM call reformulates //! the string into the formal grammar before re-attempting the parse. //! //! The formal grammar itself is UNCHANGED — rigor is preserved. //! If the engine CANNOT produce a simulation where removing A changes B, //! the edge is downgraded to "CorrelationOnly" and flagged with a warning. //! It cannot proceed through the remaining Triple-Lock stages. //! //! If |B - B'| > threshold → A causes B (Causal) /// Else → flag as CorrelationOnly pub async fn counterfactual_verify( &self, cause_var: &str, effect_var: &str, context_code: &str, // Python simulation code budget: u32, // max simulation steps ) -> Result<CounterfactualResult> { tracing::info!( "🔬 Counterfactual test: {} → {}", cause_var, effect_var ); Routing logic: //! - All models AGREE to UPVOTE → auto-accept (human notified but not required) //! - All models AGREE to DOWNVOTE → auto-reject (with lesson for MetaLearner) //! - DISAGREEMENT (|up - down| ≤ threshold) → escalate to human review //! - ANY model has confidence > spr_confidence_human_override → human review //! //! Default models: ["claude-opus-4-5", "gpt-4o", "grok-3"] //! This prevents single-model bias from automatically accepting or rejecting //! discoveries — disagreement is itself a signal of scientific interest. use anyhow::Result; use crate::config::ReviewConfig; use crate::types::{SprVote, SprVerdict, SprDecision}; use crate::llm::LlmClient; pub struct SyntheticPeerReview { cfg: ReviewConfig, clients: Vec<(String, LlmClient)>, // (model_name, client) } impl SyntheticPeerReview { pub fn new(cfg: &ReviewConfig) -> Result<Self> { let clients = cfg.spr_models.iter().map(|model| { Ok((model.clone(), LlmClient::new_with_model(model)?)) }).collect::<Result<Vec<_>>>()?; Ok(Self { cfg: cfg.clone(), clients }) } /// Run the full SPR pre-review board on a Law PR. /// /// Returns `SprDecision` with votes and routing recommendation. pub async fn review_pr( &self, pr_title: &str, equation: &str, domain: &str, r2: f64, derivation: &str, noether_charges: &[String], ) -> Result<SprDecision> { tracing::info!("🎓 SPR: reviewing PR `{pr_title}` with {} models", self.clients.len()); // Fan out to all models in parallel let vote_futures: Vec<_> = self.clients.iter().map(|(model_name, client)| { let prompt = self.build_review_prompt( pr_title, equation, domain, r2, derivation, noether_charges ); let model = model_name.clone(); let client_ref = client.clone(); async move { match client_ref.complete_json(&prompt).await { Ok(response) => Self::parse_vote(&model, &response), Err(e) => { tracing::warn!("SPR vote failed for {model}: {e}"); SprVote { model, vote: SprVerdict::Abstain, confidence: 0.0, reasoning: format!("Model unavailable: {e}"), } } } } }).collect(); let votes: Vec<SprVote> = futures::future::join_all(vote_futures).await; let decision = self.compute_decision(&votes, r2); tracing::info!( "🎓 SPR decision: {} (pre_approved={}, needs_human={})", if decision.pre_approved { "PRE-APPROVED" } else { "NEEDS REVIEW" }, decision.pre_approved, decision.needs_human ); Ok(decision) } fn build_review_prompt( &self, pr_title: &str, equation: &str, domain: &str, r2: f64, derivation: &str, noether_charges: &[String], ) -> String { format!( r#"You are a world-class theoretical physicist peer reviewer. A new physics law has been proposed by an AI research system. PR Title: {pr_title} Proposed Equation: {equation} Domain: {domain} R² Score: {r2:.4} Derivation Summary: {derivation} Noether Charges Found: {charges} Review criteria: 1. Physical meaningfulness: does this equation make sense? 2. Novelty: is this a known equation in disguise? 3. Derivation quality: are the steps logically sound? 4. Dimensional consistency: do units balance? 5. Conservation laws: does it respect known conservation principles? Return ONLY valid JSON: {{ "vote": "upvote" | "downvote" | "abstain", "confidence": 0.0-1.0, "reasoning": "2-3 sentences", "known_equivalent": "string or null", "scientific_lesson": "if downvote: lesson for MetaLearner. else null" }}"#, charges = noether_charges.join(", ") ) } fn parse_vote(model: &str, response: &str) -> SprVote { match serde_json::from_str::<serde_json::Value>(response) { Ok(v) => SprVote { model: model.to_string(), vote: match v["vote"].as_str().unwrap_or("abstain") { "upvote" => SprVerdict::Upvote, "downvote" => SprVerdict::Downvote, _ => SprVerdict::Abstain, }, confidence: v["confidence"].as_f64().unwrap_or(0.5), reasoning: v["reasoning"].as_str().unwrap_or("").to_string(), }, Err(e) => SprVote { model: model.to_string(), vote: SprVerdict::Abstain, confidence: 0.0, reasoning: format!("Parse error: {e}"), }, } } fn compute_decision(&self, votes: &[SprVote], r2: f64) -> SprDecision { let upvotes = votes.iter().filter(|v| v.vote == SprVerdict::Upvote).count() as i32; let downvotes = votes.iter().filter(|v| v.vote == SprVerdict::Downvote).count() as i32; let net_votes = upvotes - downvotes; let max_confidence = votes.iter() .map(|v| v.confidence) .fold(0.0f64, f64::max); let aggregate_score = votes.iter() .map(|v| match v.vote { SprVerdict::Upvote => v.confidence, SprVerdict::Downvote => -v.confidence, SprVerdict::Abstain => 0.0, }) .sum::<f64>() / votes.len().max(1) as f64; // Force human review if high-confidence discovery or disagreement let needs_human = net_votes.abs() <= self.cfg.spr_disagreement_threshold || max_confidence >= self.cfg.spr_confidence_human_override || r2 >= 0.99; // near-perfect fits deserve human attention let pre_approved = !needs_human && upvotes >= self.cfg.spr_quorum as i32 && downvotes == 0; SprDecision { votes: votes.to_vec(), pre_approved, needs_human, aggregate_score, } } }

v2.6 Design Principles

Efficiency is a first-class concern. The cold-start penalty of K8s jobs was acceptable in an R&D context but breaks scientific flow. The WASM fast path eliminates this friction for the majority of symbolic tasks, reducing median latency from seconds to microseconds without sacrificing isolation or rigor.

Intelligence should be guided, not exhaustive. Blind equality saturation is correct but wasteful on complex Lagrangians. The Neural Scorer Model turns brute-force exploration into directed search — the same mathematical result, reached 10-50× faster.

Correlation is not causation, and the system should know this. The Counterfactual Verification Layer prevents "interesting-sounding" but spurious relationships from polluting the knowledge graph. Only edges that survive the "What-if" test are eligible for the Triple-Lock pipeline.

A network of Lucifer instances is smarter than one. Validated laws and bridge papers gossip through the libp2p network, creating a global Scientific Hive Mind. Each instance benefits from discoveries made by all others, without sharing raw data or API keys.

Human expertise is precious; don't waste it on easy cases. The Synthetic Peer Review board handles clear-cut acceptances and rejections automatically. Human physicists see only the genuinely difficult, novel, or high-confidence discoveries — the cases that truly require human judgment.

Security must evolve with the threat model. Temporal fuzzing addresses the next generation of attacks: not breaking out of the sandbox (gVisor handles that), but using the sandbox as an oracle for timing-based side channels. Injecting noise at the µs level makes these attacks statistically infeasible.

The AI that learns should itself be versioned. Meta-MLflow ensures that the MetaLearner's evolution is as reproducible and auditable as the physics laws it helps discover. If a batch of bad lessons degrades performance, the system can roll back — a key requirement for scientific reproducibility.
